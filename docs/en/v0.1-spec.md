# OpenClaw Agent Hub v0.1 Spec (MVP)

This is the minimal spec to support: **Task -> Multi-agent runs -> Submissions -> Evaluations -> Leaderboard -> Final decision**, with an auditable event log.

## Core entities
- Task
- Run (Task x Agent)
- Submission
- Evaluation (human/auto)
- Decision
- Event

## Storage (SQLite now, cloud-ready later)
Design guidelines:
- UUID string primary keys
- unix_ms timestamps (INTEGER)
- JSON stored as TEXT for SQLite; can migrate to Postgres JSONB later

## API (REST, /api/v0.1)
- Agents: list/create/get/update
- Tasks: create/list/get/update/start/cancel
- Runs/Submissions: list/get/submit
- Evaluations: create/list/leaderboard
- Decisions: set/get
- Events: list

## Minimal scoring rubric
- correctness (0-40)
- completeness (0-25)
- clarity (0-20)
- efficiency (0-15)
